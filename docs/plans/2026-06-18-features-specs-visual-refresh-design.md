# Features 与 Specs 视觉升级设计说明

## 背景与目标

现有内容结构与信息顺序合理，但卡片数量多、视觉权重平均，整体更像功能后台而不是高端音频硬件官网。目标是在不改变文案和主要布局的前提下，提升层级、留白与产品质感。

## 方案对比

### 方案 A：克制的硬件产品风格

- Features 使用开放式分隔网格，不再堆叠独立卡片。
- Specs 使用工程参数矩阵和纵向参与路径。
- 青色只标记图标、关键值与状态。
- 优点：与当前深色官网一致，信息清晰，产品感更强。

### 方案 B：空间音频视觉化

- 加入声场轨迹、节点连线和更多动态视觉。
- 优点：表现力强；缺点：开发量和视觉负担更大。

### 方案 C：实验室终端风

- 强化命令行、状态码和等宽字体。
- 优点：技术感突出；缺点：容易偏向开发者工具。

## 推荐方案

采用方案 A。保留 Features 的 3×2 信息结构与 Specs 的左右结构，减少容器装饰，依靠分隔线、字号、留白和有限强调色建立层级。

## 详细设计

### Features

- 使用 Lucide 统一线性图标。
- 每项增加 01–06 编号。
- 使用共享分隔网格代替六张独立卡片。
- 提高标题与正文对比度，增加内部留白。

### Specs

- 左侧取消终端窗口装饰，改为参数矩阵。
- 标签保持等宽字体，参数值使用正常正文排版。
- 右侧三条参与路径使用纵向序号与分隔线。
- 四项权益合并为底部横向能力摘要。

## 测试策略

- 检查单列、双列和三列断点。
- 检查中文长标题与参数值不会溢出。
- 运行 ESLint 与 Next.js 生产构建。
- 保持 reduced-motion 兼容。
