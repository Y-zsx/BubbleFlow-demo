# 移动端导航与场景适配设计

## 背景与目标

单页分区的“下一屏”按钮在较高区块中可能无法正确识别当前位置，移动端 WebGL 场景也沿用了桌面相机参数。目标是让所有分区跳转稳定，并确保主要视觉在手机竖屏和横屏中不被无意裁切。

## 推荐方案

- 以视口中的固定位置计算当前分区，不依赖区块可见面积比例。
- 点击时重新计算当前位置，再滚动到下一个真实存在的分区。
- 统一设置固定导航的滚动偏移。
- WebGL 相机根据画布宽度切换位置和视野，首屏场景同时缩放主体。
- 手机端使用更适合空间场景的画布比例，并补齐安全区与 44px 触控目标。

## 边界处理

- 用户启用减少动态效果时使用即时滚动。
- 最后一个分区隐藏下一屏按钮。
- 画布尺寸变化时重新计算相机，覆盖旋转屏幕和窗口缩放。
- 页面最低支持 320px 宽度，不允许产生横向滚动。

## 验证

- 检查所有导航链接和下一屏按钮的目标分区。
- 检查 375px 竖屏、手机横屏与桌面视口。
- 运行 ESLint 和 Next.js 生产构建。
